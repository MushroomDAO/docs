## GToken Pricing Curve

GToken starts cheap — it is a protocol participation voucher, not an investment.

**Starting price: $0.15 USD / GT**
**Price increase per phase: +12% (fixed, on-chain)**
**Personal cap per round: 1% of that round's revenue target**

| Round | Phase | Revenue Target | GT Price | Per-Person Cap |
|---|---|---|---|---|
| Tier 1 | Powered by Love | $1,200 | $0.1500 | $12 (~80 GT) |
| Tier 2 | CMUBRC Research | $8,000 | $0.1680 | $80 (~476 GT) |
| Tier 3 | Protocol Community | $36,600 | $0.1882 | $336 (~1,786 GT) |
| Tier 4 | AAStar Infra | $86,400 | $0.2107 | $864 (~4,100 GT) |
| Tier 5+ | Ecosystem | Admin-set | +12%/tier | 1% rule |

**Price milestone trajectory:** $0.15 → $0.168 → $0.188 → $0.211

### Why Revenue-Based Tiers (Not Token-Quantity)

Funding goals are denominated in USD cost (not token count). Using revenue as the tier trigger ensures:
- Tier advancement only happens when real funding needs are met
- Impossible to game by buying tiny amounts without covering the protocol's needs
- Price discovery mirrors actual protocol value growth

## What GToken Is (and Is Not)

GToken is a **protocol participation credential** — not a financial instrument.

**What GToken gives you:**

1. Mint your SBT → cross-community reputation on Mycelium Protocol
2. Register a community, run your own Paymaster (using community points or global PNTs)
3. Build and operate your own SuperPaymaster node
4. Access the Task Plaza, voting system, and exchange services
5. Priority access to other protocols built on Mycelium
6. Launch your own protocol within the ecosystem
7. Governance rights: propose MIPs (Mycelium Improvement Proposals), vote on protocol configuration changes (1 GT = 1 vote; quadratic voting for select decisions)

**What GToken does NOT give you:** No financial return, no dividend, no guaranteed value. GToken is a governance tool, not a profit share.

### Burn Mechanism

Every join and exit from the protocol burns a defined amount of GToken. The burn rate is configurable by community multisig. Combined with a 21M hard cap, this creates natural scarcity without inflation games.

## Guardian + Invitation System

No whitelist addresses to collect. Privacy preserved.

```
You (Founder)
  └─ Invite Guardians (high-reputation people you know personally)
       └─ Each Guardian invites up to 10 people  [Level 1 airdrop]
            └─ Each L1 invites up to 5 people   [Level 2 airdrop]
                 └─ Each L2 invites up to 3      [Level 3 airdrop]
                      └─ Each L3 invites 1       [Level 4 airdrop]
```

- No address collection → no privacy leak
- Social proof verification → anti-sybil by design
- Natural scarcity at each level
- All 5 sale phases are open to the public; Guardian network handles organic spread

## Two Value Coordinate Systems

### System A — Problem × Action

```
Action Level
    ↑
    │                              ● Phase 5: AAStar Infra (continuous)
    │                    ●           Phase 4: Mycelium Community
    │          ●                     Phase 3: CMUBRC Research
    │    ●                           Phase 2: Powered by Love
    ●                                Phase 1: Voice (done)
    └────────────────────────────────────→  Problem / Value Dimension
```

Each real-world problem drives one action leap. Each leap maps to a protocol milestone and a funding target.

### System B — Output × Cost

```
Cost (USD)
    ↑
86,400 │                                  ● Phase 5 (AAStar Infra)
       │
36,600 │                   ●               Phase 4 (Community)
       │
 8,000 │         ●                         Phase 3 (Research)
       │
 1,200 │   ●                               Phase 2 (Basics)
       └────────────────────────────────────→
              Output (papers / protocol / SDK / community / DApp)
```

Cost grows non-linearly with output — but marginal cost per unit of output *decreases* as infrastructure is amortized across more users.

**GToken sales curve is isomorphic to both systems:**
- Each funding target = one action node in System A
- Funding received = cost covered in System B
- +12% price = on-chain expression of protocol value increase
